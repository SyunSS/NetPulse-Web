## 视频测试引擎 — 当前方案

### 测试流程（3 步）

```
1. DNS 探测 (DnsEngine::resolve)
   ↓
2. HTTP/TCP 探测 (HttpEngine::probe)  
   ↓
3. 浏览器打开页面 → 注入 JS 监听 <video> 播放事件 → 等待播放 → 采集数据
```

### 核心方法

用 `headless_chrome` crate 操控 Chromium，通过注入 JavaScript 来监听和采集 `<video>` 播放数据，**不真正解码视频流**。

### 采集的指标

| 指标 | 采集方式 |
|------|----------|
| 首次播放时延 | JS 监听 `video.addEventListener('playing', ...)` 记录 `performance.now()` |
| 播放是否成功 | `playing` 事件触发 或 `video.currentTime > 0` → `play_success = true` |
| 缓冲次数 | JS 监听 `video.addEventListener('waiting', ...)` 累加计数 |
| 缓冲总时间 | 每次 `waiting` → `playing` 的时间差累计 `total_buffer_time_ms` |
| 丢帧 | `video.webkitDroppedFrameCount` |
| 解码帧 | `video.webkitDecodedFrameCount` |
| 视频大小 | `performance.getEntriesByType('resource')` 过滤 `video/mp4/m3u8` 等 |
| 视频时长 | `video.duration * 1000` |
| 下载速率 | `video_size / first_play_time_ms` 推算，**不是真实 TCP 测速** |
| 页面标题 | `document.title` |
| 截图 | CDP `Page.captureScreenshot` |

### video 元素定位（5 级 fallback）

```
1. 平台配置的 video_selector（如 B 站：".bilibili-player-video video"）
   ↓ 找不到
2. 'video' 标签
   ↓ 找不到
3. '.video video' / '#video' / 'video.html5-main-video'
   ↓ 找不到
4. document.querySelectorAll('video')[0]   ← 兜底取第一个
   ↓ 找不到
5. 返回 { play_success: false, error: 'no_video_element' }
```

### 平台配置示例 (`VideoPlatformConfig`)

```toml
[[video.platforms]]
name = "Bilibili"
detect_only = false          # false = 完整播放检测
wait_seconds = 3             # 页面加载后等待秒数
video_selector = ".bilibili-player-video video"

[[video.platforms]]
name = "Netflix"
detect_only = true           # true = 仅检测页面是否可访问
```

### 已知局限

| 问题 | 影响 | 原因 |
|------|------|------|
| **DRM 保护** | Netflix/腾讯视频等无法获取播放数据 | 浏览器拒绝注入 JS 访问 DRM 内容 |
| **非 HTML5 播放器** | Flash/Silverlight 完全无法检测 | `document.querySelector('video')` 返回 null |
| **async JS evaluate** | `tab.evaluate(js, true)` 对 Promise 处理不稳定 | 和之前浏览器引擎的「首屏时延」问题同根 |
| **首帧时延不准** | 实际播放时间可能更早或更晚 | `video.play()` 是异步的，`playing` 事件受网络+解码延迟影响 |
| **下载速率不准** | `video_size / first_play_time` 只是估算 | 没有真实的 TCP 下载测速 |
| **丢帧为 0** | headless 模式下不渲染画面 | Chromium 不 gpu 渲染，无丢帧统计 |
| **不支持直播流** | HLS/DASH 直播无 duration | `video.duration` 返回 Infinity |

### 和浏览器引擎一样的问题

视频引擎也是 `async` IIFE 注入 JS（用了 `setTimeout`、`video.play().catch()`），通过 `tab.evaluate(js, true)` 等 Promise。headless_chrome 对异步 evaluate 的处理不稳定，可能导致：

- `first_play_time_ms` = `null`
- `play_success` = `false`（实际已播放）
- `buffer_count` = 0（实际有缓冲）

**建议**：改成同步 JS + 分阶段采集（先注入监听 → sleep 等待 → 同步查询结果），和刚修好的浏览器引擎一样。
